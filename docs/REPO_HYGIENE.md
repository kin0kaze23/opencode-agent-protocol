# Repository Hygiene Policy

> **Purpose:** Defines what belongs in the repo, what must be ignored, and how to maintain a clean public surface.

## Machine-Specific Path Policy

- **No hardcoded user paths** in active code or public docs
- Use `$HOME`, `$WORKSPACE_ROOT`, or relative paths
- Use `os.environ["ROOT_DIR"]` in Python heredocs
- Use `Path(__file__)` or `__dirname` for self-referencing paths
- The `verify-environment.sh` script checks for hardcoded paths

## Generated Artifact Policy

- Generated files are `.gitignored` and not tracked
- Scripts must create parent directories with `mkdir -p` before writing
- Scripts must tolerate missing generated files (bootstrap behavior)
- Curated reports (scorecards, audit reports) are tracked and kept
- Protocol Atlas rendered SVGs are tracked (they are documentation, not generated)

## Archive Policy

- `.opencode/archive/` is `.gitignored` (historical, not needed for operation)
- Canonical protocol history lives in `vault/protocols/opencode/`
- The vault CHANGELOG is the authoritative detailed history
- The root CHANGELOG.md is the public-facing summary

## Pre-Release Hygiene Checklist

Before tagging a release:

1. [ ] No `/Users/` paths in active `.opencode/` files
2. [ ] No `/Users/` paths in public docs
3. [ ] No secrets in tracked files (gitleaks passes)
4. [ ] Generated artifacts are not tracked
5. [ ] Machine-local configs are not tracked
6. [ ] Protocol Atlas version matches NOW.md version
7. [ ] All version-bearing files are coherent
8. [ ] Vault snapshot, CHANGELOG, VERSIONS.md updated
9. [ ] All conformance suites pass
10. [ ] Fresh clone would work (no local dependencies)
