## Invite Message

Subject: External review request — OpenCode Agent Protocol (15–30 min)

---

Hi [Name],

I'm building [OpenCode Agent Protocol](https://github.com/kin0kaze23/opencode-agent-protocol) — a safety-first AI engineering harness for governed agentic development. It provides risk classification, model routing, privacy scanning, protocol conformance tests, and branch protection for AI-assisted development.

I'm looking for 3–5 technical reviewers to try cloning and validating the repo, and give honest feedback on clarity, safety, and usability.

**What to test:**
- Clone the repo and run validation scripts
- Read the Capability Catalog and Runtime Map
- Inspect the Protocol Atlas (visual system map)
- Review example workflows
- Check claims, failure modes, and threat model

**Expected time:** 15–30 minutes

**How to start:**
1. Follow the [First Run Checklist](https://github.com/kin0kaze23/opencode-agent-protocol/blob/main/docs/FIRST_RUN_CHECKLIST.md)
2. Read the [External Review Guide](https://github.com/kin0kaze23/opencode-agent-protocol/blob/main/docs/EXTERNAL_REVIEW_GUIDE.md)
3. File feedback using the "External Review Feedback" issue template

**What I'm looking for:**
- Install blockers or validation failures
- Documentation that is unclear without private context
- Safety gaps or concerns
- Missing capabilities you expected
- Claims that feel overstated or understated

**What this is not:**
- Not a product codebase — it is a protocol layer
- Not a comparative benchmark — no claims of being "better than" other tools
- Not a feature request session — focus on clarity and safety

Your feedback will be triaged and summarized publicly (without personal data). See [Feedback Triage Policy](https://github.com/kin0kaze23/opencode-agent-protocol/blob/main/docs/FEEDBACK_TRIAGE.md) for details.

Thanks for considering!

---

## How to Use This Template

1. Copy the message above
2. Personalize the greeting
3. Send to 3–5 technical reviewers
4. Track responses in a private spreadsheet
5. Triage feedback per [docs/FEEDBACK_TRIAGE.md](FEEDBACK_TRIAGE.md)
